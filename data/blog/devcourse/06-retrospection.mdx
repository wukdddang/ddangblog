# 타임라인

## 데브코스에 들어가기 전

대학원 석사 과정 중, 연구실에서 여러 국가 R&D 프로젝트를 경험했었습니다. 보통 프로젝트는 컨소시엄을 꾸려 진행하게 되는데, 그때 다른 회사들과 함께 일하면서
이전에도 어렴풋이 알고 있긴 했지만, 개발자라는 직업에 대해 좀 더 관심이 갔습니다. <br/> 프로그램 / 애플리케이션을 만들다 보면 오류도 많이 발생하고, 그것이 갑작스러운 기획 변경에 막혀 버려지기도 하고, 
다시 만들어야 하는 일이 발생할지라도, 사용자가 원하는 동작을 제공해주는 개발자는, 제겐 정말 정말 매력적인 직업으로 다가왔습니다.

![안돼 안바꿔줘 돌아가](/static/images/devCourse/06/return.jpeg) 

(개발자님 기획이 바뀌어서... 프로그램을 다시 만들어야 할 것 같네요 🤔)

결론적으로 제가 되고 싶었던 모습의 직장인은 박사가 되는게 아니라, 개발자였구나 라는 생각을 그제서야 하게 되었습니다. ㅎㅎ 꽤 늦긴 했죠. <br/>
C나 C++, Python의 경우에는 연구실에서 사용했어서 문법 정도는 알고 다른 사람이 작성한 코드는 읽을 수 있는 수준이었습니다만 CS 지식은 전무한 상태였고, 
정확히 어떤 개발자가 있는지도 모르고, 어떤 분야의 개발을 하고 싶은지도 잘 몰랐기에, 일단 CS 지식은 반드시 필요할테니 졸업 전에 기본적인 CS 지식만이라도 챙겨서 나가자고 생각했습니다. <br/>

그러고 대망의 23년 2월에 졸업을 했습니다. 졸업을 하고 다시 CS 지식을 복습하면서, 어떤 분야의 개발을 하고 싶은지를 생각해보았습니다. 다른 개발 직군도 장점이 있고 재미있는 부분이 많았지만,
사용자와의 상호작용이 많은 프론트엔드 개발이 제게는 더 재미있을 것 같다는 생각이 들었습니다. 그리고 눈에 바로 보이니 더 재미있기도 했구요 ㅎㅎ<br/> 그때가 3월 초였던 것 같아요. 그때부터 JavaScript 문법을 다지기 시작했습니다.

## 1~2주차
- 데브코스 기간동안 팀이 3번 꾸려진다고 하는데 2달간 함께할 첫 팀이 발표되었습니다. 다행히 다들 개발에 열정적이고 상냥한 분들이었습니다.
- JavaScript의 기본 동작 원리를 먼저 학습했습니다. 실행 컨텍스트와 클로저, 프로토타입, 객체지향에 대해 학습했습니다.
- JavaScript를 활용하여 기본적인 자료구조 / 알고리즘의 개념을 학습하였습니다.

## 3~5주차
- Git과 Github으로 협업하는 방식에 대해 배웠습니다.
- 콜백 함수, Promise, async/await의 사용법에 대해 배웠습니다.
- 컴포넌트 방식으로 웹 페이지를 구성하는 방법에 익숙해지는 학습을 했습니다.
- Vanilla JS만을 사용하여 간단한 SPA(Single Page Application)을 구축해보았습니다.

# 좋았던 점
- 거의 매주 개인 과제가 주어지며, `Github PR`을 통해 팀원들과 서로의 코드를 리뷰했습니다. 다른 사람의 코드를 읽고 리뷰 요청을 받은 내 코드도 다시 보면서, 가독성이 좋은 코드는 무엇인지, 더 좋은 코드는
어떻게 작성할 수 있는지를 계속 고민하게 되었습니다. 이런 과정을 통해서 팀원의 코드를 클론하기도 하면서 더 많은 것을 배울 수 있었습니다.
- **Vanilla JS**를 활용해서 컴포넌트 방식으로 웹 애플리케이션을 만드는 것을 처음 해봤습니다. 컴포넌트를 직접 렌더링 하는게 아닌 상태만 변경해주는 코드를 작성하려고 하니, 
컴포넌트 단위로 개발하는 것이 더 익숙해지는 것 같았습니다.
- **Vanilla JS**로 웹 애플리케이션 개발 과제를 할 때 `eslint`, `prettier`, `webpack`을 사용해서 프론트엔드 개발 환경 세팅을 직접 해보았습니다. 
물론 도구들의 모든 기능을 사용한 것은 아니지만, 기본적인 세팅을 직접 해보면서 `React`로 작업할 때 무심코 사용했던 `CRA`의 세팅이 어떤식으로 이루어지는 지를 배울 수 있었습니다.
